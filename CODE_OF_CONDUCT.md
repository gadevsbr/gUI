# Code of Conduct

## Purpose

gUI is an open source project for building and discussing a performance-first UI runtime. The goal of this community is to keep technical collaboration direct, respectful, and useful.

## Expected behavior

When participating in issues, pull requests, discussions, commits, reviews, and documentation, contributors are expected to:

- discuss ideas, code, tradeoffs, and bugs in good faith
- focus criticism on technical decisions, not on people
- be clear, concrete, and respectful when disagreeing
- assume incomplete information before assuming bad intent
- help keep reports reproducible and actionable
- respect different experience levels and communication styles

## Unacceptable behavior

The following behavior is not acceptable:

- harassment, intimidation, threats, or personal attacks
- hate speech or discriminatory language
- repeated bad-faith arguments, trolling, or deliberate disruption
- sharing private or sensitive information without permission
- hostile review behavior that blocks collaboration without technical justification
- spam, phishing, or unrelated promotion

## Project-specific expectations

Because gUI is a runtime/framework project, discussions often involve architecture tradeoffs. Strong technical disagreement is acceptable. Personal hostility is not.

Maintain the bar for technical discussion:

- bring examples, code, benchmarks, or concrete failure cases when possible
- distinguish facts from preferences
- avoid framing style preferences as correctness issues
- keep performance claims tied to observable behavior

## Enforcement

Project maintainers may take any action they believe is necessary to protect the project and its contributors, including:

- asking for tone or behavior changes
- editing, hiding, or locking comments
- closing issues or pull requests that are no longer constructive
- temporarily or permanently restricting participation

## Reporting

If you experience or witness unacceptable behavior, contact the maintainer privately through GitHub or by email at `gadevs2020@gmail.com`.

Include:

- a link to the relevant thread, issue, pull request, or commit
- a short description of what happened
- any context that would help with review

## Scope

This code of conduct applies across the repository and project spaces, including:

- issues
- pull requests
- wiki pages
- documentation changes
- review comments
